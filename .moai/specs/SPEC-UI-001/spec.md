## 1. 개요

### 1.1 배경

현재 `streamlit_app.py`는 사주 계산 결과를 최소한으로만 표시한다. 사이드바 입력 폼에서 생년월일·시각·성별을 입력받아 API를 호출한 후, 4기둥(간지 텍스트)과 대운 표만 보여준다.

SPEC-CALC-001 구현 완료 후 API 응답에는 다음 추가 필드가 포함된다:

- `jijanggan` — 각 지지의 지장간(여기·중기·정기)
- `yuksin` — 일간 기준 각 글자의 십성(十星)
- `ohang_ratio` — 목·화·토·금·수 오행 분포 비율(%)
- `sibiunsung` — 일간 기준 각 지지의 십이운성
- `shinsal` — 신살 목록(역마살·도화살·화개살 등)
- `sewun` — 현재 연도 ±5년 세운 목록
- `pillar_meanings` — 각 기둥의 상징 의미

이 데이터를 활용하여 사주 초보자도 직관적으로 이해할 수 있는 탭 기반 대시보드로 UI를 전면 개편한다.

### 1.2 범위

본 SPEC이 포함하는 변경:

- `streamlit_app.py` 전면 재작성: 탭 기반 4패널 대시보드 구현
- 입력 폼 개선: 시주 미상 처리 방식 명확화, 음력·윤달 UX 개선
- 4개 탭 신규 구현:
  - 탭 1 — 사주 원국 (4기둥 + 상징 의미)
  - 탭 2 — 십성 분석 (십성 테이블 + 의미 설명)
  - 탭 3 — 운의 흐름 (대운 + 세운 통합 뷰)
  - 탭 4 — 세부 지표 (지장간, 십이운성, 신살, 오행 차트)

본 SPEC이 포함하지 않는 사항:

- API 라우터 또는 도메인 로직 변경 (SPEC-CALC-001 담당)
- 인증·세션 관리 (별도 SPEC으로 분리)
- 모바일 전용 레이아웃 (데스크톱 우선 구현)
- 사용자 데이터 저장·히스토리 기능

### 1.3 목표

1. **가독성**: 사주 초보자가 각 항목의 의미를 즉시 이해할 수 있도록 설명 텍스트를 함께 표시한다.
2. **완결성**: SPEC-CALC-001이 제공하는 모든 계산 결과를 UI에서 빠짐없이 표시한다.
3. **일관성**: Streamlit 네이티브 컴포넌트만 사용하여 외부 CSS 의존성을 최소화한다.
4. **견고성**: 시주 미상, 신살 없음, 세운 데이터 없음 등 예외 상황을 우아하게 처리한다.

### 1.4 선행 조건

- **SPEC-CALC-001 구현 완료 필수**: API 응답에 `jijanggan`, `yuksin`, `ohang_ratio`, `sibiunsung`, `shinsal`, `sewun`, `pillar_meanings` 필드가 포함되어야 한다.
- SPEC-API-001 및 SPEC-CORE-001은 이미 완료 상태이다.

## 3. 가정 (Assumptions)

1. SPEC-CALC-001 구현 완료 후 API 응답 구조가 확정된다. 본 SPEC은 해당 응답 구조를 기반으로 작성되었다.
2. `pillar_meanings`는 연주·월주·일주·시주 각각에 대해 한국어 설명 문자열을 반환한다.
3. `shinsal`이 빈 리스트인 경우 "해당 없음" 메시지를 표시한다.
4. `sewun` 목록은 현재 연도를 포함한 ±5년, 총 최대 11개 항목을 반환한다.
5. 오행 비율(`ohang_ratio`)은 목·화·토·금·수 키에 대한 퍼센트 값(0~100) 딕셔너리이다.
6. 십성(`yuksin`)은 각 기둥의 천간·지지에 대한 한자+한글 표기를 포함한다.

## 4. 요구사항 (Requirements)

### 4.1 입력 폼 (REQ-UI-001 ~ REQ-UI-005)

**REQ-UI-001** [Ubiquitous]
시스템은 항상 사이드바에서 생년월일(년/월/일), 출생 시각, 성별, 양력/음력 구분을 입력받아야 한다.

**REQ-UI-002** [Event-Driven]
WHEN 사용자가 "음력" 옵션을 선택하면 THEN 윤달 체크박스가 사이드바에 표시되어야 한다.

**REQ-UI-003** [State-Driven]
IF 사용자가 "시각 미상" 체크박스를 선택한 상태이면 THEN 시각 슬라이더는 비활성화되고 시주는 미상으로 처리되어야 한다.

**REQ-UI-004** [Unwanted]
시스템은 1600년 이전 또는 2100년 이후 연도를 유효한 입력으로 받아서는 안 된다.

**REQ-UI-005** [Event-Driven]
WHEN 사용자가 "사주 계산" 버튼을 클릭하면 THEN 로딩 스피너가 표시되며 API 호출이 완료될 때까지 입력 영역이 잠겨야 한다.

### 4.2 사주 원국 탭 (REQ-UI-010 ~ REQ-UI-015)

**REQ-UI-010** [Ubiquitous]
시스템은 항상 4기둥(年柱·月柱·日柱·時柱)을 천간과 지지가 분리된 카드 형태로 표시해야 한다.

**REQ-UI-011** [Ubiquitous]
시스템은 항상 각 기둥 아래에 해당 기둥의 상징 의미(조상/어린시절, 부모/직업환경, 나/배우자, 자녀/말년)를 설명 텍스트로 표시해야 한다.

**REQ-UI-012** [State-Driven]
IF 시주 데이터가 없는(미상) 상태이면 THEN 시주 카드에는 간지 대신 "미상" 텍스트와 함께 설명 의미(자녀/말년)만 표시해야 한다.

**REQ-UI-013** [Ubiquitous]
시스템은 항상 4기둥 아래에 기본 정보 섹션을 표시해야 한다. 기본 정보는 절입일(양력 환산), 대운 방향, 대운수(시작 나이)를 포함한다.

**REQ-UI-014** [Optional]
가능하면 각 기둥의 천간·지지를 큰 폰트 크기로 강조하여 가독성을 높여야 한다.

### 4.3 십성 분석 탭 (REQ-UI-020 ~ REQ-UI-025)

**REQ-UI-020** [Ubiquitous]
시스템은 항상 일간(日干)을 기준으로 나머지 7글자(년간·년지·월간·월지·일지·시간·시지)의 십성을 테이블로 표시해야 한다.

**REQ-UI-021** [Ubiquitous]
시스템은 항상 십성 테이블에 십성명(한자+한글), 음양, 오행, 소속 기둥을 열로 포함해야 한다.

**REQ-UI-022** [Ubiquitous]
시스템은 항상 비겁·식상·재성·관성·인성의 5개 그룹별로 성격과 사회적 의미를 요약 설명해야 한다.

**REQ-UI-023** [State-Driven]
IF 시주가 미상이면 THEN 시간·시지에 해당하는 십성 행은 "미상"으로 표시되어야 한다.

**REQ-UI-024** [Event-Driven]
WHEN 사용자가 특정 십성 행을 클릭하거나 해당 그룹 설명을 펼치면 THEN 해당 십성의 상세 설명이 확장 영역에 표시되어야 한다. (가능한 경우 `st.expander` 활용)

### 4.4 운의 흐름 탭 (REQ-UI-030 ~ REQ-UI-036)

**REQ-UI-030** [Ubiquitous]
시스템은 항상 대운 목록을 나이·대운간지·십성 순서로 테이블에 표시해야 한다.

**REQ-UI-031** [Ubiquitous]
시스템은 항상 현재 대운(현재 나이가 포함된 대운)을 시각적으로 강조해야 한다.

**REQ-UI-032** [Ubiquitous]
시스템은 항상 세운 목록을 연도·세운간지·대운간지 순서로 테이블에 표시해야 한다.

**REQ-UI-033** [Ubiquitous]
시스템은 항상 현재 연도(세운)를 시각적으로 강조해야 한다.

**REQ-UI-034** [State-Driven]
IF API 응답에 세운 데이터가 없거나 빈 리스트이면 THEN "세운 데이터가 없습니다" 안내 메시지를 표시해야 한다.

**REQ-UI-035** [Ubiquitous]
시스템은 항상 대운과 세운 섹션 사이에 현재 대운·세운 요약 정보를 카드 형태로 표시해야 한다.

### 4.5 세부 지표 탭 (REQ-UI-040 ~ REQ-UI-048)

**REQ-UI-040** [Ubiquitous]
시스템은 항상 지장간을 4개 지지별로 여기·중기·정기 3열 테이블로 표시해야 한다.

**REQ-UI-041** [Ubiquitous]
시스템은 항상 십이운성을 일간 기준으로 각 지지(년지·월지·일지·시지)에 대한 12단계 이름(장생·목욕·관대·건록·제왕·쇠·병·사·묘·절·태·양)과 함께 표시해야 한다.

**REQ-UI-042** [State-Driven]
IF 시주가 미상이면 THEN 시지 십이운성 셀은 "미상"으로 표시되어야 한다.

**REQ-UI-043** [Ubiquitous]
시스템은 항상 신살 목록을 이름·설명 2열 테이블로 표시해야 한다.

**REQ-UI-044** [State-Driven]
IF 신살 목록이 비어 있으면 THEN "해당 신살이 없습니다" 안내 메시지를 표시해야 한다.

**REQ-UI-045** [Ubiquitous]
시스템은 항상 오행 분포를 막대 차트로 시각화해야 한다. 차트는 목·화·토·금·수를 X축으로, 비율(%)을 Y축으로 표시한다.

**REQ-UI-046** [Optional]
가능하면 오행 차트에서 가장 강한 오행과 가장 약한 오행을 텍스트로 요약하여 차트 아래에 표시해야 한다.

**REQ-UI-047** [Unwanted]
시스템은 지장간·십이운성·신살 데이터가 API 응답에 없을 때 KeyError나 AttributeError를 사용자에게 노출해서는 안 된다.

### 4.6 오류 처리 (REQ-UI-050 ~ REQ-UI-053)

**REQ-UI-050** [Event-Driven]
WHEN API 호출이 타임아웃되거나 네트워크 오류가 발생하면 THEN 오류 메시지와 함께 재시도 안내를 표시해야 한다.

**REQ-UI-051** [Event-Driven]
WHEN API가 400 상태코드를 반환하면 THEN 입력값 검증 실패 메시지를 한국어로 표시해야 한다.

**REQ-UI-052** [Event-Driven]
WHEN API가 500 상태코드를 반환하면 THEN 서버 오류 메시지와 함께 관리자 문의 안내를 표시해야 한다.

**REQ-UI-053** [Unwanted]
시스템은 예외 스택 트레이스를 사용자에게 직접 노출해서는 안 된다.

## 5. 사양 (Specifications)

### 5.1 UI 구조

```
streamlit_app.py
├── 사이드바 (입력 폼)
│   ├── 달력 타입 선택 (양력 / 음력)
│   ├── 생년월일 입력 (년/월/일 3열)
│   ├── 윤달 체크박스 (음력 선택 시만 표시)
│   ├── 시각 입력 (슬라이더 + 시각 미상 체크박스)
│   ├── 성별 선택 (남성 / 여성)
│   └── "사주 계산" 버튼
│
└── 메인 영역
    ├── [계산 전] 안내 메시지
    └── [계산 후] 탭 4개
        ├── Tab 1: 사주 원국
        │   ├── 4기둥 카드 (年·月·日·時)
        │   ├── 각 기둥 상징 의미
        │   └── 기본 정보 (절입일, 대운 방향, 대운수)
        ├── Tab 2: 십성 분석
        │   ├── 일간 정보 헤더
        │   ├── 십성 테이블 (7글자)
        │   └── 5그룹 설명 (expander)
        ├── Tab 3: 운의 흐름
        │   ├── 현재 대운/세운 요약 카드
        │   ├── 대운 테이블
        │   └── 세운 테이블
        └── Tab 4: 세부 지표
            ├── 지장간 테이블
            ├── 십이운성 테이블
            ├── 신살 테이블
            └── 오행 분포 차트
```
