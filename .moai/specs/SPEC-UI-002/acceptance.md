# SPEC-UI-002 인수 기준: 합충형해파 Streamlit UI 표시

## 인수 기준 개요

모든 인수 기준은 `uv run streamlit run streamlit_app.py` 실행 후 "세부 지표" 탭에서 검증한다.

---

## 시나리오 1: 합충 관계가 있는 사주 결과 표시

**요구사항 참조**: R-001, R-002

```gherkin
Given  사주 계산 결과에 하나 이상의 합충형해파 관계가 포함되어 있고
       result.hapchung 리스트가 비어 있지 않을 때
When   사용자가 "세부 지표" 탭을 클릭하면
Then   "합충형해파 (合沖刑害破)" 섹션 제목이 오행 섹션 아래에 표시되고
And    테이블에 모든 관계가 행으로 나타나고
And    테이블 열은 "기둥1", "지지1", "기둥2", "지지2", "관계", "세부유형" 순서로 표시된다
```

**검증 항목**:
- [ ] 섹션 제목 "합충형해파 (合沖刑害破)" 존재
- [ ] 테이블이 화면에 표시됨
- [ ] 행 수 = `result.hapchung` 리스트 길이
- [ ] 모든 6개 열 (기둥1, 지지1, 기둥2, 지지2, 관계, 세부유형) 존재
- [ ] 각 관계의 pillar1, ji1, pillar2, ji2, relation_type 값이 올바르게 표시됨

## 시나리오 2: 합충 관계가 없는 사주 결과 처리

**요구사항 참조**: R-003

```gherkin
Given  사주 계산 결과의 hapchung 리스트가 비어 있을 때
When   사용자가 "세부 지표" 탭을 클릭하면
Then   테이블이 표시되지 않고
And    "기둥 간 특별한 관계가 없습니다." 안내 메시지가 표시된다
```

**검증 항목**:
- [ ] 테이블 컴포넌트 미표시
- [ ] "기둥 간 특별한 관계가 없습니다." 메시지 표시 (st.info 스타일)
- [ ] 에러 발생 없음

## 시나리오 3: 충(沖) 관계 행 색상 강조

**요구사항 참조**: R-004

```gherkin
Given  사주 계산 결과에 "충" 관계 유형을 포함한 hapchung 항목이 있고
       다른 관계 유형(형, 해, 파, 육합 등)도 함께 존재할 때
When   합충형해파 테이블이 표시되면
Then   "충" 관계 행의 배경색이 붉은 계열로 강조되고
And    다른 관계 행의 배경색은 기본값으로 유지된다
```

**검증 항목**:
- [ ] "충" 관계 행에 붉은 계열 배경색 적용
- [ ] "형", "해", "파", "육합", "삼합", "방합" 행은 기본 색상 유지
- [ ] 색상 강조가 관계 열 값에만 의존함 (다른 기준 없음)

## 시나리오 4: 형(刑) 세부유형 표시

**요구사항 참조**: R-002 (세부유형 열)

```gherkin
Given  hapchung 리스트에 relation_type이 "형"이고
       subtype이 "시세지형"인 HapchungRelation이 포함되어 있을 때
When   합충형해파 테이블이 표시되면
Then   해당 행의 "세부유형" 열에 "시세지형"이 표시된다
```

**검증 항목**:
- [ ] `subtype`이 있는 형 관계: "세부유형" 열에 값 표시
- [ ] `subtype`이 없는 관계 (충, 해, 파, 육합 등): "세부유형" 열이 빈 문자열

**세부유형 형식 규칙**:

| subtype 값 | 세부유형 열 표시 |
|------------|----------------|
| "시세지형" | 시세지형 |
| "무은지형" | 무은지형 |
| "무례지형" | 무례지형 |
| "자형" | 자형 |
| None 또는 "" | (빈 문자열) |

## 시나리오 5: 용어 설명 expander (선택적)

**요구사항 참조**: R-005

```gherkin
Given  합충형해파 섹션이 화면에 표시되어 있을 때
When   사용자가 "합충형해파 용어 설명" expander를 클릭하면
Then   충, 형, 해, 파, 육합, 삼합, 방합 각 관계 유형의 설명이 표시된다
And    expander의 기본 상태는 접힌 상태(closed)이다
```

**검증 항목**:
- [ ] "합충형해파 용어 설명" expander 존재
- [ ] 기본 상태: 접힘 (expanded=False)
- [ ] 클릭 시 7가지 관계 유형 설명 표시
- [ ] 각 설명에 한자 병기 포함 (예: 충(沖), 형(刑))

## 엣지케이스 검증

### EC-01: hapchung 필드가 None인 경우

```gherkin
Given  result 객체에서 hapchung 속성이 None일 때
When   render_tab_detail()가 호출되면
Then   에러 없이 "기둥 간 특별한 관계가 없습니다." 메시지가 표시된다
```

**검증 항목**:
- [ ] AttributeError 발생 없음
- [ ] TypeError 발생 없음
- [ ] 안내 메시지 정상 표시

### EC-02: 단일 관계만 존재하는 경우

```gherkin
Given  hapchung 리스트에 정확히 1개의 관계만 있을 때
When   테이블이 표시되면
Then   1행짜리 테이블이 정상적으로 표시된다
```

### EC-03: 모든 관계 유형이 동시에 존재하는 경우

```gherkin
Given  hapchung 리스트에 충, 형, 해, 파, 육합, 삼합, 방합이 모두 포함될 때
When   테이블이 표시되면
Then   모든 행이 올바른 관계 유형으로 표시되고
And    "충" 관계 행만 붉은색으로 강조된다
```
