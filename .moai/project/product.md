# 사주 분석 웹 서비스 - 제품 명세서

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 사주 분석 웹 서비스 (Python 리디자인) |
| 버전 | 0.6.0 |
| 기반 | 기존 manse_ori (Node.js) 재설계 |
| 작성일 | 2026-02-27 |

### 프로젝트 설명

한국 전통 사주팔자(四柱八字) 분석 엔진을 Python으로 재구현한 웹 서비스입니다.

생년월일시(生年月日時)와 이름, 성별을 입력받아 사주팔자를 계산하고, OpenAI GPT-4o를 활용하여 이해하기 쉬운 한국어 운세 해설을 자동으로 생성합니다.

기존 Node.js 기반의 manse_ori를 Python 생태계로 전환하여 코드 품질, 유지보수성, 확장성을 크게 향상시킵니다.

## 핵심 기능 목록

### 1. 양음력 변환 (Korean Calendar Conversion)

- 양력(그레고리력) ↔ 음력(태음력) 상호 변환
- 400년치 절기 데이터(julgi.json) 기반 정확한 변환
- 윤달(閏月) 처리 지원

### 2. 사주팔자 계산 (Four Pillars of Destiny)

- 년주(年柱): 태어난 해의 천간(天干)·지지(地支)
- 월주(月柱): 절기(節氣) 기준 월의 천간·지지
- 일주(日柱): 태어난 날의 천간·지지 (일간이 자신을 상징)
- 시주(時柱): 태어난 시각의 천간·지지

### 3. 절기 탐지 (Solar Term Detection)

- 24절기(二十四節氣) 정확한 시각 계산
- 절입(節入) 시각 기준 월주 결정
- julgi.json의 400년 절기 데이터 활용

### 4. 지장간 추출 (Hidden Stems)

- 각 지지(地支) 내부에 감춰진 천간(天干) 추출
- 여기(餘氣), 중기(中氣), 정기(正氣) 3단계 구분
- 사주 심층 분석의 핵심 요소

### 5. 음양오행 분석 (Yin-Yang & Five Elements)

- 음양(陰陽): 각 간지의 음·양 판별
- 오행(五行): 목(木)·화(火)·토(土)·금(金)·수(水) 분류
- 오행 비율 및 균형 분석

### 6. 육신 분석 (Relationship to Day Master)

- 비겁(比劫): 비견(比肩), 겁재(劫財)
- 식상(食傷): 식신(食神), 상관(傷官)
- 재성(財星): 편재(偏財), 정재(正財)
- 관성(官星): 편관(偏官), 정관(正官)
- 인성(印星): 편인(偏印), 정인(正印)

### 7. 합충 분석 (Harmony & Conflict Analysis)

- 삼합(三合): 신자진(申子辰), 해묘미(亥卯未), 인오술(寅午戌), 사유축(巳酉丑)
- 육합(六合): 자축(子丑), 인해(寅亥), 묘술(卯戌), 진유(辰酉), 사신(巳申), 오미(午未)
- 충(冲): 자오충(子午冲), 축미충(丑未冲) 등 6가지
- 형(刑): 지세지형(持勢之刑), 무은지형(無恩之刑), 무례지형(無禮之刑)
- 해(害): 6가지 해 관계

### 8. 용신 분석 (Use God Analysis)

- 일간(日干) 강약(强弱) 판단
- 억부용신(抑扶用神): 강한 일간 억제 또는 약한 일간 부조
- 조후용신(調候用神): 계절적 기운 균형 조절
- 통관용신(通關用神): 대립하는 오행 사이 중재

### 9. 대운 계산 (Major Periods)

- 대운수(大運數) 계산: 절기까지의 날수 기반
- 10년 주기 대운 전개
- 대운 천간지지 및 오행 특성

### 10. OpenAI LLM 통합 (Fortune Explanation)

- GPT-4o 기반 한국어 운세 해설 자동 생성
- 사주 분석 결과를 구조화된 프롬프트로 변환
- 초보자도 이해할 수 있는 친절한 설명체

## 주요 사용 사례 (Use Cases)

### UC-001: 기본 사주 조회

```
행위자: 일반 사용자
전제조건: 사용자가 생년월일시를 알고 있음
주 흐름:
  1. 사용자가 이름, 생년월일시, 성별을 입력
  2. 시스템이 사주팔자를 계산
  3. 시스템이 육신, 합충, 용신을 분석
  4. OpenAI가 한국어 운세 해설 생성
  5. 결과 화면에 표시
```

### UC-002: API를 통한 사주 데이터 조회

```
행위자: 개발자
전제조건: API 키 인증 완료
주 흐름:
  1. POST /api/v1/saju 엔드포인트 호출
  2. JSON 형식으로 생년월일시, 이름, 성별 전달
  3. 사주 계산 결과 JSON 응답 수신
  4. LLM 해설 포함 여부 선택 가능
```

### UC-003: 음력 생일 입력

```
행위자: 일반 사용자
전제조건: 사용자가 음력 생년월일을 알고 있음
주 흐름:
  1. 사용자가 음력 입력 모드 선택
  2. 음력 생년월일 입력 (윤달 여부 포함)
  3. 시스템이 양력으로 변환
  4. 사주 계산 진행
```

### UC-004: 대운 흐름 조회

```
행위자: 일반 사용자
전제조건: 사주 조회 완료
주 흐름:
  1. 사주 결과 화면에서 대운 탭 선택
  2. 현재 대운 및 향후 대운 표시
  3. 각 대운의 오행 특성 확인
  4. OpenAI가 대운별 운세 흐름 해설
```

## 기존 manse_ori 대비 개선 사항

| 개선 항목 | 기존 manse_ori (Node.js) | 새 버전 (Python) |
|-----------|--------------------------|------------------|
| 전역 변수 사용 | 광범위한 전역 변수 의존 | 요청 범위(Request-scoped) 객체로 격리 |
| 타입 안전성 | JavaScript 동적 타입, 런타임 오류 위험 | Python 타입 힌트 + Pydantic 검증 |
| 도메인 모델 | 비정형 딕셔너리 전달 | 데이터클래스/Pydantic 모델 명확한 구조 |
| 비동기 처리 | 콜백 기반 비동기 | async/await 명확한 비동기 패턴 |
| 에러 처리 | 비일관적 에러 처리 | 계층적 예외 처리 및 검증 |
| 테스트 | 60+ 케이스 일부 분산 | pytest 기반 통합 테스트 스위트 |
| API 문서화 | 수동 문서 | FastAPI 자동 OpenAPI/Swagger 문서 |
| LLM 통합 | 없음 | OpenAI GPT-4o 운세 해설 생성 |
| UI | 없음 (API 전용) | Streamlit 프로토타입 UI |
| 팩토리 패턴 | 없음 | 출력 인터페이스 팩토리 패턴 |

## 제약 사항 및 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 응답 시간 | 사주 계산 < 500ms, LLM 해설 포함 < 10s |
| 지원 브라우저 | Streamlit UI 기준 Chrome, Firefox, Safari 최신 버전 |
| 입력 범위 | 1600년 ~ 2100년 사이의 생년월일 |
| 언어 | 한국어 전용 (운세 해설) |
| OpenAI 의존성 | LLM 해설은 선택사항, API 없이도 기본 결과 제공 |
| 데이터 보안 | 개인 생년월일 데이터 서버 저장 안 함 |
